NameVisitor
ClassificationBehavioural
StrategyDelegation
IntentRepresent an operation to be performed on the elements of an object structure. Visitor lets you define a new operation without chaning the classes of the elements on which it operates.

Participants

  • Visitor
    • Each class of ConcreteElemnt has a visit() operation declared for it.
    • The operation’s signature identifies the class that sends the visit() request to the visitor.
    • The particular class is then accessed through the interface defined for it.
  • ConcreteVisitor
    • Implements the operations defined by visitor.
    • May store information about objects that are visited.
  • Element
    • Defines an accept() operation that takes an object of Visitor as a parameter.
  • ConcreteElement
    • Implements an accept() operation that takes an object of Visitor as a parameter.
  • ObjectStructure
    • Has a highlevel interface that allows the visitor access and traverse its elements.
    • The structure may be a composite of a collection such as an array, list or a set.
